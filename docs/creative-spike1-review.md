# User needs 

## Code/design review Creative Spike week 1 

Deze week hebben jullie gewerkt aan de eerste creative coding spike. Voor de opdracht van de opdrachtgever heb je een ontwerp gemaakt voor creatieve oplossingen voor de interface. Deze ben je vervolgens gaan bouwen met (één van) de JS libraries van deze week (`GSAP` en `Threlte.xyz`). 

### Aanpak

Vandaag gaan jullie de creative spikes van elkaar bekijken en aantal daarvan gaan jullie verder onderzoeken. Je presenteert allereerst jouw gemaakte werk in een marktsetting. Studenten en docenten lopen bij iedereen langs en mogen stemmen op de meest creatieve, complexe en vetste spike die is gemaakt. Je mag drie keer een stem uitbrengen. Vervolgens worden de repo's met de meeste stemmen gedeeld in _Teams_ en ga je in een duo de code bekijken en daar een geleerde les over delen in het _random_ kanaal. 

### Presenteren van je werk: 

#### Voorbereiding van de presentatie

Zet jouw werk klaar voor de creative coding spike markt! Open een browser en maak daar drie tabjes aan: 
- Eentje voor het eindresultaat
- Een andere voor jouw inspiratie (moodboard, figma, schets..)
- En tenslotte open je je code in GitHub van de creative coding spike

Voor jouw laptop leg je een A4 papiertje neer en schrijf bovenaan het blad je naam en welke woorden combinatie jij hebt gebruikt om tot deze coding spike te komen. 

#### Stemmen

Vervolgens loop je rond en kijk wat jouw peers hebben gemaakt. Laat je inspireren! Je mag drie keer stemmen. Dit doe je door jouw naam op het blaadje te noteren van een peer. Je mag niet op jezelf stemmen en de drie stemmen mogen niet bij één persoon worden genoteerd. 

#### Reviewen van de repo's met de meeste stemmen

Na het tellen van de stemmen worden de repo's met de meeste votes in Teams gedeeld. 

Vorm een duo en kies samen twee repo's uit die jullie gaan reviewen. Pak de code erbij. Hoe is deze creative coding spike gemaakt? Wat valt jullie op? Schrijf in het random kanaal samen één bericht over wat jullie is opgevallen. Schrijf een mini tutorial hoe iemand anders ook met deze techniek kan werken en voeg eventueel een codepen toe ter verduidelijking. Let ook op hoe het component RAP/PE is gemaakt. Dit kan een onderwerp zijn om op in te zoomen. 

### Volgende week
Elke week maken jullie een nieuwe creative spike. Je begint elke week met een leeg canvas. Gebruik de vrijdagen om inspiratie op te doen bij peers. Dit kun je meenemen in het ontwerpen van een nieuwe creative spike volgende week. Ook kun je de vrijdagen gebruiken om een user test uit te voeren met een peer. Neem opmerkingen op in het pull-request. Voer opgedane inzichten door en voeg die als commits toe aan het pull-request.
